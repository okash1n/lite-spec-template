# Lite Spec 運用ガイド

`.litespec/` は、仕様・技術判断・運用ルールを軽量に管理するためのディレクトリです。
このファイルはテンプレート利用後も残し、運用ルールの参照先として使います。

## ファイルの役割

- `.litespec/SPEC.md`: 全体仕様のインデックス（概要とリンク）
- `.litespec/specs/`: 機能単位の詳細仕様
- `.litespec/adr/`: 重要な技術判断の記録
- `AGENTS.md`: 人と AI エージェント向けの作業ルール（リポジトリ直下）
- `README.md`: プロジェクト全体の説明（リポジトリ直下）

## 運用フロー

1. 作業前に `.litespec/SPEC.md` を確認する
2. 対象機能の `.litespec/specs/*.md` を確認する
3. 仕様に影響する変更では `SPEC.md` と関連する詳細仕様を更新する
4. 重要な判断は `.litespec/adr/` に記録する
5. 実装後に仕様と実装の乖離がないか確認する

## おすすめの使い方

### 既存仕様や関連ドキュメントがある場合

1. 既存ドキュメントを LLM に渡す
2. `AGENTS.md` を読ませたうえで `.litespec/` への反映を依頼する
3. 生成結果を見て `SPEC.md` と `specs/` のリンクを整える

プロンプト例:

```text
このリポジトリの AGENTS.md を読んだうえで、添付ドキュメントの内容を
.litespec/SPEC.md と .litespec/specs/ に反映してください。
全体方針は SPEC.md、機能詳細は specs に分割し、
不足している前提や未確定事項は TODO として明記してください。
```

### まだ仕様が固まっていない場合

1. `AGENTS.md` を基準に、初期仕様作成の質問を LLM に依頼する
2. 一問一答で要件を固める
3. 回答結果を `SPEC.md` と `specs/` に起こす

プロンプト例:

```text
このリポジトリの AGENTS.md を読んで、
このプロジェクトの初期仕様を固めるための質問を
ユーザーに一問一答形式で投げかけてください。
質問は優先度順に 1 つずつ提示し、各回答を要約してから次へ進んでください。
最後に .litespec/SPEC.md と .litespec/specs/ に反映できる形で
「確定事項」と「未確定事項」を分けて出力してください。
```

## テンプレート利用時の注意

- 新規プロジェクト作成直後に `README.md` をプロジェクト内容へ書き換える
- `AGENTS.md` は固定セクションを維持し、必要なルールは「プロジェクト追記セクション」に追加する
- 運用ルールが固まったら `AGENTS.md` のプロジェクト追記セクションを更新する
- `.litespec/` は初期雛形として残し、必要に応じて `specs/` を追加する
