# lite-spec-template

Kiro ほど重くない「仕様管理の最低限セット」を使い回すためのテンプレートです。
新しいプロジェクトの直下に配置して、仕様・意思決定・運用ルールを最小コストで揃えることを目的にしています。

## 使い方

1. このリポジトリをテンプレートとして新規プロジェクトを作成する
2. 既存プロジェクトにファイルをコピーして使う

## 含まれるもの

- `SPEC.md`: 1ページ仕様（目的/範囲/要件/DoD など）
- `AGENTS.md`: 人・AIエージェント共通の運用ルール
- `docs/adr/0001-template.md`: ADR（意思決定ログ）テンプレート

## 運用の考え方

- **SPEC は契約書**: 目的/範囲/DoD を常に最新に保つ
- **ADR は判断の記録**: 重要な技術判断は必ず残す
- **AGENTS は現場のルール**: 実装/レビュー/ドキュメント方針を揃える

## 推奨フロー

1. `SPEC.md` を埋める
2. `AGENTS.md` をプロジェクトに合わせて調整
3. 重要な判断が出たら `docs/adr/` に記録

## 参考

- このテンプレートは「軽量な共通言語」を狙っています
- Kiro の思想は活かしつつ、運用コストを抑える設計です
